Terrain as Log-Likelihood

The connection between walking uphill and fitting a statistical model

The Metaphor You've Been Using

In the optimisation pages, you watched algorithms climb a terrain — gradient ascent following the steepest slope, Newton-Raphson using curvature to leap toward the peak, simulated annealing randomly wandering to escape local hills.

That terrain wasn't just a convenient visual metaphor. In statistics, the terrain is the log-likelihood surface. Every point on the map represents a candidate set of model parameters, and the elevation at that point is how well those parameters explain the observed data.

Terrain Concept Statistical Concept
Map coordinates (x, y) Parameter values ($\beta_0, \beta_1, \ldots$)
Elevation at a point Log-likelihood $\ell(\beta | \text{data})$
Peak / summit Maximum likelihood estimate (MLE)
Slope (gradient) Score function $\nabla \ell(\beta)$
Curvature (Hessian) Observed information matrix $-\nabla^2 \ell(\beta)$
Confidence ellipse at peak Approximate confidence region for $\beta$
Sharp peak vs broad plateau Small vs large standard errors
"Climbing a hill" is maximum likelihood estimation. The algorithms are the same — only the surface they're climbing changes.

Why Different Models Make Different Surfaces

Gaussian (Normal) Regression

The log-likelihood is a perfect quadratic bowl (an upside-down paraboloid). This is why Newton-Raphson converges in exactly one step — the quadratic approximation is the function. The Hessian is constant everywhere.

$\ell(\beta_0, \beta_1) = -\frac{1}{2\sigma^2} \sum_{i=1}^{n} (y_i - \beta_0 - \beta_1 x_i)^2 + \text{const}$

Logistic Regression

The log-likelihood is curved but not quadratic — it bends more in some directions than others, and the curvature changes as you move. Newton-Raphson still converges quickly, but needs multiple steps because its quadratic approximation is only locally accurate.

$\ell(\beta_0, \beta_1) = \sum_{i=1}^{n} \left[ y_i \log p_i + (1-y_i) \log(1-p_i) \right]$ where $p_i = \frac{1}{1 + e^{-(\beta_0 + \beta_1 x_i)}}$

Poisson Regression

The log-likelihood surface can be more elongated and asymmetric, especially with skewed count data. The curvature depends on the predicted counts, creating a surface that is steeper in some directions than others.

$\ell(\beta_0, \beta_1) = \sum_{i=1}^{n} \left[ y_i (\beta_0 + \beta_1 x_i) - e^{\beta_0 + \beta_1 x_i} \right] + \text{const}$

The shape of the log-likelihood surface is determined by the model family. Gaussian gives a bowl. Logistic gives a curved hill. Poisson gives an asymmetric ridge. But the peak of each surface is the MLE.
