Two Models, One Likelihood

Every previous tutorial fitted a single model with a single linear predictor. The ZIP model is different: it contains two complete sub-models, each with its own predictors, link function, and coefficients.

Zero-Inflation Component

A logistic regression predicting whether person $i$ is a structural zero:

$\pi_i = \text{logit}^{-1}(z_i\gamma)$

Predictors $z_i$: has_insurance, distance_to_clinic. Coefficients: $\gamma$ (3 parameters).

Count Component

A Poisson regression predicting the visit count for the "at risk" population:

$\mu_i = \exp(x_i\beta)$

Predictors $x_i$: age, income, chronic_condition. Coefficients: $\beta$ (4 parameters).

For our doctor visits model that means seven parameters ($\gamma_0, \gamma_1, \gamma_2$ and $\beta_0, \beta_1, \beta_2, \beta_3$) must all be estimated at once, because they jointly determine the probability of each observed count.

Why IRLS Doesn't Apply Here

In Tutorials 1–5 the fitting workhorse was IRLS (iteratively reweighted least squares). IRLS works because a standard GLM has one linear predictor feeding one exponential-family distribution — each iteration is just a weighted regression.

ZIP breaks both requirements: it has two linear predictors, and the mixture $\pi + (1-\pi)e^{-\mu}$ is not an exponential-family distribution. There is no single weighted regression that updates everything. We need a different strategy.

The Chicken-and-Egg Problem

Suppose we somehow knew which of the zeros were structural. Fitting would be easy:

  • Fit a logistic regression of "is structural zero?" on has_insurance and distance_to_clinic → gives $\gamma$
  • Fit a Poisson regression on everyone else → gives $\beta$

But the labels are hidden. A person with zero visits might be structurally excluded, or might simply have drawn a 0 from their Poisson distribution. We can't tell by looking at the data — that's exactly the membership the model is supposed to estimate.

The EM Idea: Guess, Fit, Improve, Repeat

The Expectation-Maximisation (EM) algorithm resolves the circularity by alternating between soft guesses about the hidden labels and ordinary model fits:

  1. Initialise: start with rough values for $\gamma$ and $\beta$

    e.g. fit a plain Poisson for $\beta$ and a logistic regression on "is zero?" for $\gamma$

  2. E-step (soft assignment): for each observation with $y_i = 0$, compute the probability it is a structural zero, given the current parameters:
    $w_i = \dfrac{\pi_i}{\pi_i + (1 - \pi_i)\,e^{-\mu_i}}$

    Observations with $y_i > 0$ get $w_i = 0$: a positive count proves they are in the at-risk population

  3. M-step (fit each component): using the $w_i$ as weights:
    • update $\gamma$ by a weighted logistic regression (each zero counts as $w_i$ "structural" and $1-w_i$ "at risk")
    • update $\beta$ by a weighted Poisson regression on the at-risk portion (weights $1-w_i$)

    Both M-step fits are ordinary GLMs — so IRLS does reappear, but only inside each component

  4. Repeat E and M steps until the parameters stop changing

    Each cycle is guaranteed never to decrease the log-likelihood

The Intuition

EM turns one impossible problem ("fit a mixture") into two easy problems ("fit a logistic regression" and "fit a Poisson regression") connected by a soft labelling step. A zero from an uninsured person living 25 km from the clinic gets a high $w_i$ (probably structural); a zero from an insured, healthy 25-year-old gets a lower $w_i$ (plausibly just a quiet year). As the parameters improve, the soft labels improve, and vice versa.

What Software Actually Does

EM is the clearest way to understand mixture fitting, but most ZIP implementations (including R's pscl::zeroinfl and Python's statsmodels) take a more direct route: they hand the joint log-likelihood of all seven parameters to a general-purpose optimiser such as BFGS or Newton-Raphson, which climbs the 7-dimensional likelihood surface using gradients.

For our doctor visits data, pscl::zeroinfl converges in 14 BFGS iterations to a log-likelihood of −752.7. Direct maximisation is usually faster than EM, and the curvature (Hessian) at the peak provides the standard errors for free.
