Choose TWO Link Functions

A zero-inflated model has two components, each needing its own link function. First select the link for the count model (left), then the link for the zero-inflation model (right).

Part 1: Count Model Link Select

For the Poisson count component: how many visits do non-structural-zero patients make?

Log Link

$g(\mu) = \log(\mu)$

Same log link as Poisson (Tutorial 3). Ensures positive count predictions.

Identity Link

$g(\mu) = \mu$

Predictions equal the linear predictor directly. Allows negative counts.

Part 2: Zero-Inflation Link Select

For the structural-zero logistic component: what is the probability of being a structural zero?

Logit Link

$g(\pi) = \log\left(\frac{\pi}{1-\pi}\right)$

Same logit link as logistic regression (Tutorial 2). Models probability of being a structural zero.

Identity Link

$g(\pi) = \pi$

Probability not bounded to (0,1). Can predict impossible probabilities.

✓ Both Link Functions Selected

Count model: Log link — ensures positive count predictions.
Zero-inflation model: Logit link — bounds structural-zero probability to (0, 1).

This is the first model with two link functions — one for each component. The count part uses log (like Poisson), the zero-inflation part uses logit (like logistic regression). Two models in one.

✔ Correct! Log link for the count component.

The log link is the standard choice for count models because:

Inverse link: $\mu = e^{\eta}$ is always positive.

Counts can't be negative, so the log link guarantees valid predictions.

This is identical to the link used in standard Poisson regression (Tutorial 3). The count component of a ZIP model is just a regular Poisson model applied to the "at-risk" population.

❌ Wrong! Identity link allows negative counts.

With the identity link, predictions equal the linear predictor:

$\mu = \eta = \beta_0 + \beta_1 X_1 + ...$

This can easily be negative, which makes no sense for counts!

The count component needs the log link to ensure positive predictions, just like standard Poisson regression.

✔ Correct! Logit link for the zero-inflation component.

The logit link is perfect for modelling probabilities because:

Inverse link: $\pi = \frac{e^{\eta}}{1 + e^{\eta}}$ is always between 0 and 1.

The probability of being a structural zero is bounded to a valid range.

This is identical to the link used in logistic regression (Tutorial 2). The zero-inflation component is effectively a logistic regression predicting whether someone is a "structural zero" (never visits) or "at risk" (might visit).

❌ Wrong! Identity link gives unbounded probabilities.

With the identity link, the predicted probability equals the linear predictor:

$\pi = \eta = \gamma_0 + \gamma_1 Z_1 + ...$

This can give $\pi > 1$ or $\pi < 0$ — impossible for a probability!

The zero-inflation component models a probability, so it needs the logit link to keep predictions in the valid (0, 1) range, just like logistic regression.
