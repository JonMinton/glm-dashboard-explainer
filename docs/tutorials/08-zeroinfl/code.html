The Two-Part Formula

Because ZIP contains two sub-models, the model formula needs two parts. R's pscl::zeroinfl separates them with a vertical bar |:

doctor_visits ~ age + income_10k + chronic_condition|has_insurance + distance_to_clinic
Left of the bar: predictors for the count component (Poisson, log link) — how many visits, for the at-risk population.
Right of the bar: predictors for the zero-inflation component (logistic) — the probability of being a structural zero.

The two predictor sets can overlap, but here they are completely different: health factors drive how many visits, while access factors (insurance, distance) drive whether visiting is possible at all.

A Practical Tip: Rescale Big Predictors

We use income_10k = income / 10000 rather than raw income. With income on its raw scale (15,000–120,000) the likelihood surface is so stretched in one direction that the curvature matrix becomes numerically singular — the optimiser converges but the standard errors come back as NA. Rescaling fixes this; only the coefficient's units change.

# Load required libraries
library(pscl)   # zeroinfl() and vuong()
library(MASS)   # glm.nb() for comparison

# Load the doctor visits dataset
visits_data <- read.csv("doctor_visits.csv")

# Rescale income for numerical stability
visits_data$income_10k <- visits_data$income / 10000

# Fit Zero-Inflated Poisson
# Formula has two parts separated by |
#   Left of |:  count model predictors (Poisson, log link)
#   Right of |: zero-inflation predictors (logistic)
fit_zip <- zeroinfl(doctor_visits ~ age + income_10k + chronic_condition |
                    has_insurance + distance_to_clinic,
                    data = visits_data, dist = "poisson")

summary(fit_zip)

# Fit the simpler models we rejected on the Distribution page
fit_poisson <- glm(doctor_visits ~ age + income_10k + chronic_condition,
                   family = poisson(link = "log"), data = visits_data)
fit_nb <- glm.nb(doctor_visits ~ age + income_10k + chronic_condition,
                 data = visits_data)

# Compare all three by AIC (lower is better)
AIC(fit_poisson, fit_nb, fit_zip)

# Formal test for zero-inflation
vuong(fit_zip, fit_poisson)
import pandas as pd
import statsmodels.api as sm
from statsmodels.discrete.count_model import ZeroInflatedPoisson
from statsmodels.discrete.discrete_model import NegativeBinomial

# Load the doctor visits dataset
visits_data = pd.read_csv("doctor_visits.csv")

# Rescale income for numerical stability
visits_data['income_10k'] = visits_data['income'] / 10000

y = visits_data['doctor_visits']

# exog: count model design matrix; exog_infl: zero-inflation design matrix
X = sm.add_constant(visits_data[['age', 'income_10k', 'chronic_condition']])
Z = sm.add_constant(visits_data[['has_insurance', 'distance_to_clinic']])

# Fit ZIP (method='bfgs' is more reliable here than the default)
zip_model = ZeroInflatedPoisson(endog=y, exog=X, exog_infl=Z)
fit_zip = zip_model.fit(method='bfgs', maxiter=500)
print(fit_zip.summary())

# Comparison models
fit_poisson = sm.Poisson(y, X).fit()
fit_nb = NegativeBinomial(y, X).fit(method='bfgs', maxiter=500)

# AIC comparison -- compute ZIP's by hand: statsmodels' reported AIC
# for ZIP omits the inflation parameters from its parameter count
k_zip = 7  # 4 count params + 3 inflation params
aic_zip = -2 * fit_zip.llf + 2 * k_zip
print(f"AIC  Poisson: {fit_poisson.aic:.1f}")
print(f"AIC  NegBin:  {fit_nb.aic:.1f}")
print(f"AIC  ZIP:     {aic_zip:.1f}")

R and Python Agree

The Python fit converges to the same solution: log-likelihood −752.70 with identical coefficients (e.g. inflate_has_insurance = −1.2273, chronic_condition = 0.5587). In the statsmodels summary the zero-inflation coefficients appear first, prefixed with inflate_. statsmodels may print a convergence warning even though the optimiser has in fact reached the same maximum as R — check that the reported log-likelihood matches.

The Key Result: ZIP Wins the Model Comparison

AIC drops from 1659.9 (Poisson) to 1586.2 (NegBin) to 1519.4 (ZIP) — exactly the diagnostic journey from the Distribution page, now in numbers. The Vuong test confirms it formally: z = 5.81, p = 3.1×10−9, decisively favouring ZIP over plain Poisson.

Note that ZIP wins despite being penalised for two extra parameters — AIC already accounts for model complexity. The improvement comes from capturing the right structure, not from extra flexibility alone.

Interpreting the Two Sets of Coefficients

Count component (log link — same interpretation as Tutorials 3 and 4):

  • exp(0.559) = 1.75: a chronic condition raises the expected number of visits by about 75%, among people who can visit
  • exp(0.0098) = 1.010: each additional year of age increases expected visits by about 1%

Zero-inflation component (logit link — same interpretation as Tutorial 2, but predicting structural-zero membership):

  • exp(−1.227) = 0.29: having insurance cuts the odds of being a structural zero by 71%
  • exp(0.031) = 1.031: each extra kilometre to the clinic raises the odds of being a structural zero by about 3%

The payoff: two different stories from one model. Health drives how many visits; access drives whether visits happen at all. A single-component model would blur these together.
