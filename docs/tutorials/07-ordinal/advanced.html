The Latent Variable View

The cleanest way to understand the proportional odds model is through a latent (unobserved) continuous variable. Imagine each patient has a true underlying pain intensity $Y^*$ on a continuous scale:

$$Y_i^* = \mathbf{x}_i'\boldsymbol{\beta} + \varepsilon_i, \qquad \varepsilon_i \sim \text{standard logistic}$$

We never observe $Y^*$ directly. Instead, the patient reports the category their latent pain falls into, determined by the thresholds (cut-points) $\alpha_1 < \alpha_2 < \alpha_3$:

$$Y_i = j \quad \iff \quad \alpha_{j-1} < Y_i^* \le \alpha_j$$

with the conventions $\alpha_0 = -\infty$ and $\alpha_J = +\infty$. Because the standard logistic distribution has CDF $F(z) = \text{logit}^{-1}(z) = 1/(1 + e^{-z})$, this latent construction gives exactly the cumulative logit model:

$$P(Y_i \le j) = P(\varepsilon_i \le \alpha_j - \mathbf{x}_i'\boldsymbol{\beta}) = \text{logit}^{-1}(\alpha_j - \mathbf{x}_i'\boldsymbol{\beta})$$

Why the Thresholds Are Free Parameters

The latent scale has no natural units — "pain intensity" cannot be measured in metres or millimetres of mercury. So the model fixes the error distribution (standard logistic, scale 1) and lets the data choose where the cut-points sit. The fitted thresholds tell us how the four labels carve up the latent scale: a wide gap between $\hat{\alpha}_1$ and $\hat{\alpha}_2$ means "Mild" covers a broad band of underlying intensities. This is also why there is no separate intercept: adding a constant to $\mathbf{x}'\boldsymbol{\beta}$ and to every $\alpha_j$ leaves all probabilities unchanged, so the thresholds are the intercepts.

Swap the logistic error for a standard normal and you get the ordered probit model — the same idea we met with binary probit in Tutorial 2, now with multiple cut-points instead of one.

Deriving the Log-Likelihood

We have $J = 4$ ordered categories, thresholds $\boldsymbol{\alpha} = (\alpha_1, \alpha_2, \alpha_3)$ and slopes $\boldsymbol{\beta}$. Write $\eta_i = \mathbf{x}_i'\boldsymbol{\beta}$ for the linear predictor.

Step 1: Cumulative Probabilities

The cumulative logit model specifies, for $j = 1, \ldots, J-1$:

$$P(Y_i \le j) = \text{logit}^{-1}(\alpha_j - \eta_i) = \frac{1}{1 + \exp\!\big(-(\alpha_j - \eta_i)\big)}$$

with the boundary conventions $P(Y_i \le 0) = 0$ and $P(Y_i \le J) = 1$.

Step 2: Category Probabilities by Differencing

The probability of landing exactly in category $j$ is the difference of consecutive cumulative probabilities:

$$\pi_{ij} = P(Y_i = j) = P(Y_i \le j) - P(Y_i \le j - 1) = \text{logit}^{-1}(\alpha_j - \eta_i) - \text{logit}^{-1}(\alpha_{j-1} - \eta_i)$$

The ordering constraint $\alpha_1 < \alpha_2 < \alpha_3$ guarantees every $\pi_{ij} > 0$, because $\text{logit}^{-1}$ is strictly increasing.

Step 3: Likelihood of One Observation

Each observation contributes the probability of the category it actually fell in. Using the indicator $\mathbb{1}(y_i = j)$, which is 1 when patient $i$ reported category $j$ and 0 otherwise:

$$L_i(\boldsymbol{\alpha}, \boldsymbol{\beta}) = \prod_{j=1}^{J} \pi_{ij}^{\,\mathbb{1}(y_i = j)}$$

This is one draw from a multinomial distribution — but with the $\pi_{ij}$ structured through the shared thresholds and slopes.

Step 4: Take Logs and Sum Over Observations
$$\ell(\boldsymbol{\alpha}, \boldsymbol{\beta}) = \sum_{i=1}^{n} \sum_{j=1}^{J} \mathbb{1}(y_i = j)\, \ln\!\big[ P(Y_i \le j) - P(Y_i \le j-1) \big]$$
The Full Log-Likelihood:

$$\ell(\boldsymbol{\alpha}, \boldsymbol{\beta}) = \sum_{i=1}^{n} \sum_{j=1}^{J} \mathbb{1}(y_i = j)\, \ln\!\Big[ \text{logit}^{-1}(\alpha_j - \mathbf{x}_i'\boldsymbol{\beta}) - \text{logit}^{-1}(\alpha_{j-1} - \mathbf{x}_i'\boldsymbol{\beta}) \Big]$$
$$\text{with } \alpha_0 = -\infty \;(\text{so logit}^{-1} = 0) \text{ and } \alpha_J = +\infty \;(\text{so logit}^{-1} = 1)$$

The Score for $\boldsymbol{\beta}$

Write $F(z) = \text{logit}^{-1}(z)$ for the logistic CDF and $f(z) = F(z)\big(1 - F(z)\big)$ for its density. For an observation with $y_i = j$, the chain rule gives:

$$\frac{\partial \ell_i}{\partial \boldsymbol{\beta}} = -\,\frac{f(\alpha_j - \eta_i) - f(\alpha_{j-1} - \eta_i)}{F(\alpha_j - \eta_i) - F(\alpha_{j-1} - \eta_i)}\; \mathbf{x}_i$$

Unlike the Poisson or binomial cases, this does not simplify to a tidy $\sum_i (y_i - \mu_i)\mathbf{x}_i$ form — each observation's contribution involves a ratio of density differences to CDF differences. That is why polr and OrderedModel reach for general-purpose optimisers (Newton-Raphson, BFGS) rather than the IRLS algorithm used by standard GLMs.

Keeping the Thresholds Ordered

The log-likelihood is only defined while $\alpha_1 < \alpha_2 < \alpha_3$ — cross two thresholds and some $\pi_{ij}$ goes negative, so its log is undefined. Optimisers handle this by re-parameterising: statsmodels estimates $(\alpha_1, \ln(\alpha_2 - \alpha_1), \ln(\alpha_3 - \alpha_2))$, so any unconstrained parameter values map back to validly ordered thresholds. That is exactly the "log-increment" output we decoded on the code page.

The Proportional Odds Assumption

Why is this called the proportional odds model? Take the log cumulative odds at threshold $j$ for two patients with covariates $\mathbf{x}_1$ and $\mathbf{x}_2$:

$$\text{logit}\,P(Y \le j \mid \mathbf{x}_1) - \text{logit}\,P(Y \le j \mid \mathbf{x}_2) = (\alpha_j - \mathbf{x}_1'\boldsymbol{\beta}) - (\alpha_j - \mathbf{x}_2'\boldsymbol{\beta}) = -(\mathbf{x}_1 - \mathbf{x}_2)'\boldsymbol{\beta}$$

The threshold $\alpha_j$ cancels. The log odds ratio between any two covariate profiles is the same at every threshold $j$ — the cumulative odds curves are parallel on the logit scale, differing only by a constant shift. One $\boldsymbol{\beta}$, one story, at every split of the scale.

What the Assumption Buys (and Costs)

Parsimony: with $J = 4$ categories and $p = 4$ predictors, the proportional odds model uses $3 + 4 = 7$ parameters. An unordered multinomial logit would use $(J-1)(p+1) = 15$. Fewer parameters means more precise estimates and a single interpretable odds ratio per predictor.

The cost: if a predictor genuinely affects the None/Mild split differently from the Moderate/Severe split (for example, a drug that prevents severe pain but does nothing for mild discomfort), the single-$\boldsymbol{\beta}$ model is misspecified and will average the two effects.

Checking the Assumption

A practical diagnostic is to fit $J - 1$ separate binary logistic regressions, one per split: $\mathbb{1}(Y > 1)$, $\mathbb{1}(Y > 2)$, $\mathbb{1}(Y > 3)$. Under proportional odds each predictor's coefficient should be roughly the same across all three fits. Formal versions of this comparison include:

  • Brant test (R package brant) — tests each predictor's coefficient stability across thresholds;
  • Likelihood ratio test against the unordered multinomial model — a large improvement in fit suggests the ordering constraint is too restrictive;
  • Partial proportional odds models (R package VGAM, ordinal) — relax the assumption only for the predictors that violate it.

For our synthetic pain data the assumption holds by construction — the data-generating process used one $\boldsymbol{\beta}$ at every threshold. Real data are rarely so obliging, so this check belongs in any serious ordinal analysis.

The Pattern Breaks — Productively

Tutorials 1–6 each modelled a single number per observation through one mean $\mu_i$. The ordinal model is our first example where each observation is modelled as a whole probability distribution over categories, held together by shared slopes and ordered thresholds. It stretches the GLM template — multiple intercepts, a difference of two inverse links — while keeping its core logic: a linear predictor, a link, and maximum likelihood.
