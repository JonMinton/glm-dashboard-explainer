Choose the Distribution Family

Pain level has 4 ordered categories. The response is categorical with a natural ordering. Which distribution family is appropriate for modelling the probability of each ordered category?

Cumulative (Ordinal)

$P(Y = j) = P(Y \le j) - P(Y \le j-1)$

Multinomial with ordering constraint. Probability distributed across ordered categories, with cumulative structure.

Support: $\{1, 2, \ldots, J\}$ (ordered)

Multinomial

$P(Y = j) = \pi_j, \quad \sum \pi_j = 1$

General categorical distribution. No ordering assumed — each category is independent.

Support: $\{1, 2, \ldots, J\}$ (unordered)

Gaussian (Normal)

$\text{Var}(Y) = \sigma^2$

Continuous, symmetric distribution. The default for truly continuous data.

Support: $(-\infty, \infty)$

Poisson

$\text{Var}(Y) = \mu$

For unbounded counts of events. Variance equals the mean.

Support: $\{0, 1, 2, \ldots\}$ (counts)

✓ Cumulative (Ordinal) Distribution Selected

The cumulative ordinal distribution models probabilities across ordered categories using $J-1$ thresholds and shared slopes.

✔ Correct! Cumulative ordinal uses the ordering.

The cumulative ordinal distribution is the right choice because it:

  • Respects ordering: Models $P(Y \le j)$, which only makes sense for ordered categories
  • Threshold structure: Think of a latent continuous variable cut by $J-1$ thresholds
  • Efficient: Fewer parameters than unordered multinomial (shared slopes)

The latent variable interpretation:

Imagine there's a continuous "true pain" $Y^*$ that we can't observe directly. We only see which of 4 bins it falls into:

Y*: ----[None]----$\alpha_1$----[Mild]----$\alpha_2$----[Moderate]----$\alpha_3$----[Severe]----

The thresholds $\alpha_1, \alpha_2, \alpha_3$ are the boundaries between categories. The predictors shift the location of $Y^*$, changing which category is most likely.

⚠ Valid but ignores the ordering.

The multinomial distribution works for any categorical outcome, but it doesn't use the ordering information in pain levels.

Multinomial logistic: Estimates separate parameters for each category comparison.

With 4 categories and 3 predictors: $(J-1) \times p = 3 \times 3 = 9$ slope parameters (counting the 3-level treatment as one predictor; with dummy coding it is $3 \times 4 = 12$).

Problems with ignoring ordering:

  • More parameters: 9 slopes vs 3 slopes in proportional odds
  • No proportional odds interpretation: Can't say "treatment shifts the whole distribution"
  • Less efficient: Wider confidence intervals due to more parameters
  • Harder to interpret: Each predictor has $J-1$ separate effects instead of one

When categories are ordered, using the ordering constraint gives a more powerful and more interpretable model.

❌ Wrong! Pain level is not continuous.

The Gaussian distribution is for continuous data that can take any real value.

Problem: Predicting pain = 2.7 is meaningless. There's no pain level between Mild and Moderate.

Why Gaussian doesn't work for ordinal data:

  • Pain level can only be 1, 2, 3, or 4 — not continuous
  • Residuals cannot be normally distributed for a 4-level outcome
  • Predictions can fall outside the valid range (e.g., pain = 0.3 or 5.1)
  • Assumes equal spacing between categories, which is rarely true

For ordered categories, we need a distribution that assigns probabilities to discrete ordered outcomes.

❌ Wrong! Pain levels are not counts.

The Poisson distribution is for unbounded counts of events (0, 1, 2, 3, ...).

Problem: Pain levels are categories that happen to be numbered, not counts of anything.

Key differences between counts and ordinal categories:

  • Counts are unbounded: Can be 0, 1, 2, ..., 100, etc. Pain has exactly 4 levels.
  • Counts are additive: 2 events + 3 events = 5 events. "Mild" + "Moderate" = ??? (meaningless)
  • Poisson variance = mean: This constraint makes no sense for 4 ordered categories
  • No ceiling: Poisson allows any non-negative integer; pain level is bounded at 4
