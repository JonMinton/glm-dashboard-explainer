How Proportional Odds Model Fitting Works

The proportional odds model uses maximum likelihood estimation (MLE) to find the best-fitting thresholds and slopes. Unlike standard GLMs that use IRLS, ordinal models typically use Newton-Raphson or other general optimisers.

The Parameters

With $J = 4$ pain categories and $p$ predictors, we estimate:

  • $J - 1 = 3$ threshold parameters: $\alpha_1, \alpha_2, \alpha_3$ (also called cut-points or intercepts)
  • $p$ slope parameters: $\beta_1, \ldots, \beta_p$ (shared across all thresholds)

Total: $3 + p$ parameters instead of $(J-1) \times p$ for unordered multinomial.

The Cumulative Logit Model

For each observation $i$ and threshold $j$:

$$P(Y_i \le j) = \frac{1}{1 + \exp(-(\alpha_j - \mathbf{x}_i'\boldsymbol{\beta}))}$$

where $P(Y \le 0) = 0$ and $P(Y \le J) = 1$ by convention.

Category Probabilities

The probability of each specific category is the difference between consecutive cumulative probabilities:

$$P(Y_i = j) = P(Y_i \le j) - P(Y_i \le j-1)$$

For example, with 4 categories:

  1. $P(Y = 1)$ = $P(Y \le 1) - 0$ = $P(Y \le 1)$

    Probability of "None" pain

  2. $P(Y = 2)$ = $P(Y \le 2) - P(Y \le 1)$

    Probability of "Mild" pain

  3. $P(Y = 3)$ = $P(Y \le 3) - P(Y \le 2)$

    Probability of "Moderate" pain

  4. $P(Y = 4)$ = $1 - P(Y \le 3)$

    Probability of "Severe" pain

The Log-Likelihood

MLE finds parameters that maximise the probability of the observed data:

$$\ell(\boldsymbol{\alpha}, \boldsymbol{\beta}) = \sum_{i=1}^{n} \log P(Y_i = y_i)$$

where each $P(Y_i = y_i)$ is computed from the cumulative model above.

Proportional Odds Assumption

The key assumption is that the same $\boldsymbol{\beta}$ applies at every threshold. This means each predictor shifts the entire distribution by the same amount on the logit scale.

Interpretation: $e^{\beta}$ gives the odds ratio for being in a higher vs lower category, regardless of where you split the scale.

What Software Does

R uses MASS::polr() and Python uses statsmodels.miscmodels.ordinal_model.OrderedModel. Both maximise the log-likelihood using iterative numerical methods (Newton-Raphson or BFGS).

What Makes Ordinal Special

Unlike other GLMs where there's one intercept, the ordinal model has $J-1$ intercepts (the thresholds) but still only one set of slopes. This is the proportional odds structure: many intercepts, shared slopes.

For our pain model: 3 thresholds ($\alpha_1, \alpha_2, \alpha_3$) + treatment, dose, and age coefficients.
