Choose the Link Function

Pain level has 4 ordered categories (None < Mild < Moderate < Severe). Which link function respects this ordering and models the cumulative probabilities?

Cumulative Logit

$g(P(Y \le j)) = \log\frac{P(Y \le j)}{P(Y > j)}$

Models cumulative probabilities using logit. The proportional odds model — one set of slopes $\beta$, with $J-1$ threshold intercepts.

Logit (Binary)

$g(\mu) = \log\frac{\mu}{1-\mu}$

The standard logit link for binary (two-category) outcomes.

Identity Link

$g(\mu) = \mu$

Predictions equal the linear predictor directly. Simple additive interpretation.

Log Link

$g(\mu) = \log(\mu)$

Ensures positive predictions via $\mu = e^{\eta}$. Multiplicative interpretation.

Key Insight: Multiple Thresholds, One Set of Slopes

The proportional odds assumption means $\beta$ has the same effect at every cut-point — treatment shifts the entire pain distribution, not just one comparison. Each $\beta$ has a single interpretation: it shifts the distribution up or down.

✓ Cumulative Logit Selected

The cumulative logit models ordered categories with proportional odds — one set of slopes with $J-1$ thresholds.

✔ Correct! Cumulative logit respects the ordering.

The cumulative logit (proportional odds) link is the standard choice for ordinal data because:

Model: $\text{logit}(P(Y \le j)) = \alpha_j - \mathbf{x}'\boldsymbol{\beta}$

There are $J-1 = 3$ threshold intercepts ($\alpha_1, \alpha_2, \alpha_3$) and one shared set of slopes $\boldsymbol{\beta}$.

Why this works:

  • Models cumulative probabilities $P(Y \le j)$, which respect the ordering
  • One set of $\beta$ coefficients — each coefficient shifts the entire distribution
  • $e^{\beta}$ gives the odds ratio for being in a higher vs lower category
  • Thresholds $\alpha_j$ capture the baseline cut-points between categories

Proportional odds: The effect of each predictor is the same regardless of which cut-point you're considering. Treatment B shifts the entire pain distribution, not just one threshold.

❌ Wrong! Standard logit only works for 2 categories.

The binary logit link models a single probability $P(Y = 1)$, which works for two categories (success/failure).

Problem: Pain has 4 levels — you'd need to collapse the scale to binary, losing information.

What you'd lose:

  • Collapsing "None vs Mild/Moderate/Severe" discards the ordering within the higher categories
  • Different binary splits give different answers — which one do you pick?
  • You'd need multiple separate binary models instead of one unified model

The cumulative logit generalises the binary logit to multiple ordered categories by modelling cumulative probabilities at each threshold simultaneously.

❌ Wrong! Identity treats categories as equally-spaced numbers.

The identity link means $\mu = \eta$ directly. This treats the outcome as a continuous number, implying equal spacing between categories.

Problem: "Moderate" is not twice "Mild". The gaps between categories are unknown and may differ.

Treating ordinal as continuous:

  • Assumes the difference between None (1) and Mild (2) equals Moderate (3) vs Severe (4)
  • Predicting pain = 2.7 is meaningless — there's no pain level between Mild and Moderate
  • Can predict values outside the valid range (e.g., pain = 0.3 or 5.1)

Ordinal regression avoids these assumptions by modelling cumulative probabilities rather than treating the category numbers directly.

❌ Wrong! Log link is designed for positive continuous outcomes.

The log link ensures positive predictions ($\mu = e^{\eta}$), which is useful for counts (Poisson) or positive continuous data (Gamma).

Problem: Pain levels are categories that happen to be numbered, not counts of anything.

Why log doesn't work here:

  • Pain level is not a count or a positive continuous measure
  • $e^{\eta}$ gives a continuous positive number, not a probability distribution over ordered categories
  • No mechanism for modelling transitions between ordered categories

For ordered categories, we need a link that models the cumulative probabilities $P(Y \le j)$ at each threshold between categories.
