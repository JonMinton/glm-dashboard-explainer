Fitting the Proportional Odds Model in R and Python

Our model: pain_level ~ treatment + dose_mg + age with a cumulative logit link. The dataset has $n = 350$ patients, with pain recorded as an ordered factor (None < Mild < Moderate < Severe) and treatment as a three-level factor (A, B, Placebo — A is the reference level).

R
Python
# Load required library (polr lives in MASS, not base R)
library(MASS)

# Load the pain levels dataset
pain_data <- read.csv("pain_levels.csv")

# CRUCIAL: declare the response as an ORDERED factor
pain_data$pain_level <- factor(pain_data$pain_level, levels = 1:4,
                               labels = c("None", "Mild", "Moderate", "Severe"),
                               ordered = TRUE)
pain_data$treatment <- factor(pain_data$treatment,
                              levels = c("A", "B", "Placebo"))

# Fit the proportional odds model (cumulative logit)
fit <- polr(pain_level ~ treatment + dose_mg + age,
            data = pain_data,
            method = "logistic",   # cumulative logit link
            Hess = TRUE)           # keep Hessian for standard errors

# View model summary
summary(fit)

# polr reports t values but no p-values - compute them ourselves
ctable <- coef(summary(fit))
p <- pnorm(abs(ctable[, "t value"]), lower.tail = FALSE) * 2
cbind(ctable, "p value" = round(p, 4))

# Odds ratios with confidence intervals
exp(coef(fit))
exp(confint(fit))
import pandas as pd
import numpy as np
from statsmodels.miscmodels.ordinal_model import OrderedModel

# Load the pain levels dataset
pain_data = pd.read_csv("pain_levels.csv")

# CRUCIAL: declare the response as an ORDERED categorical
pain_data["pain_cat"] = pd.Categorical(
    pain_data["pain_level"], categories=[1, 2, 3, 4], ordered=True
)

# Build the design matrix: dummy-code treatment (A = reference)
# NOTE: no intercept column - the thresholds play that role
X = pd.get_dummies(pain_data["treatment"],
                   prefix="treatment", drop_first=True).astype(float)
X["dose_mg"] = pain_data["dose_mg"].astype(float)
X["age"] = pain_data["age"].astype(float)

# Fit the proportional odds model (cumulative logit)
model = OrderedModel(pain_data["pain_cat"], X, distr="logit")
fit = model.fit(method="bfgs")

# View model summary
print(fit.summary())

# Recover the actual thresholds (statsmodels reports increments)
thresholds = model.transform_threshold_params(fit.params[-3:])
print(thresholds)  # [-inf, -2.538, -0.095, 1.773, inf]

# Odds ratios
print(np.exp(fit.params[:4]))

Gotcha: The Two Packages Report Thresholds Differently

R's polr reports the three thresholds directly as "Intercepts": $\hat{\alpha}_1 = -2.538$, $\hat{\alpha}_2 = -0.095$, $\hat{\alpha}_3 = 1.773$.

Python's OrderedModel reports the first threshold (row 1/2) directly, but the remaining rows are log-increments: $\hat{\alpha}_2 = -2.5375 + e^{0.8932} = -0.095$ and $\hat{\alpha}_3 = -0.095 + e^{0.6248} = 1.773$. This parameterisation guarantees the thresholds stay ordered during optimisation. Use transform_threshold_params() to recover the actual cut-points — both packages then agree exactly (same log-likelihood: $-404.51$).

Interpreting the Output

Remember the model: $\text{logit}\, P(Y \le j) = \alpha_j - \mathbf{x}'\boldsymbol{\beta}$. A positive $\beta$ pushes probability towards higher (worse) pain categories; a negative $\beta$ pushes towards lower pain. Because of proportional odds, $e^{\beta}$ is a single odds ratio that applies at every threshold.

Term Estimate Odds Ratio $e^{\hat\beta}$ 95% CI Interpretation
treatmentB −1.153 0.316 (0.199, 0.496) Treatment B cuts the odds of being in a higher pain category to about a third of Treatment A's, at any threshold
treatmentPlacebo 0.198 1.220 (0.608, 2.456) Placebo patients lean towards more pain than Treatment A, but the CI comfortably spans 1 — no clear effect (note: placebo patients also receive zero dose)
dose_mg −0.0173 0.983 (0.975, 0.991) Each extra mg of dose multiplies the odds of higher pain by 0.983 — about 16% lower odds per 10 mg ($0.983^{10} \approx 0.84$)
age 0.0213 1.021 (1.007, 1.036) Each extra year of age raises the odds of higher pain by about 2%

Sanity Check Against the True Parameters

This dataset is synthetic, so we know the data-generating truth: thresholds $(-2.0, 0.5, 2.5)$ and slopes $\beta_B = -0.8$, $\beta_{\text{Placebo}} = 0.6$, $\beta_{\text{dose}} = -0.02$, $\beta_{\text{age}} = 0.03$.

The fitted values $(-1.15, 0.20, -0.017, 0.021)$ have the correct signs and plausible magnitudes, and every 95% CI on the natural scale is consistent with the truth. The placebo effect is hardest to pin down ($n = 68$ placebo patients, and placebo is entangled with dose $= 0$) — exactly the kind of honest uncertainty a wide CI should report.

Predicting Category Probabilities

For a 50-year-old patient on Treatment A with a 50 mg dose, the linear predictor is $\mathbf{x}'\hat{\boldsymbol{\beta}} = -0.0173 \times 50 + 0.0213 \times 50 \approx 0.20$, giving:

Pain Level Calculation Probability
None $\text{logit}^{-1}(-2.538 - 0.20)$ 0.061
Mild $\text{logit}^{-1}(-0.095 - 0.20) - 0.061$ 0.366
Moderate $\text{logit}^{-1}(1.773 - 0.20) - 0.427$ 0.401
Severe $1 - \text{logit}^{-1}(1.773 - 0.20)$ 0.172

In R: predict(fit, newdata, type = "probs"). In Python: fit.predict(X_new) returns one column per category. The four probabilities always sum to 1 — the model distributes one unit of probability across the ordered categories.

Why Not Just lm() / OLS on 1–4?

A linear model on the numeric codes would happily predict "pain = 2.7" or "pain = 4.3", and it assumes the None→Mild gap equals the Moderate→Severe gap. The ordinal model makes neither mistake: it only ever produces a proper probability distribution over the four actual categories, and it lets the fitted thresholds determine how "wide" each category is.

Where do the Std. Error columns in these outputs come from? From the curvature of the log-likelihood at its peak — see Standard Errors from Curvature.
